LazyHTMLv1.0.0

A tiny runtime by RoomOff (Jade Hamel) that reads styling straight off your HTML tags — color, spacing, layout, hover states, breakpoints — and lazily resolves them into a single shared CSS class per element. No build step, no stylesheet to maintain, no class-naming decisions.

Quick start

Drop the bundle on the page. LazyHTML scans the DOM on load and keeps watching it after.

htmlindex.html
<script src="dist/lazy-html.js"></script>

<button
  bg="#34d399" color="#04150e"
  pad-x="20" pad-y="10" radius="999"
  bold="true"
  hover:scale="1.03" transition="all .15s ease"
>
  Ship it
</button>

That's it — no class name, no CSS file. LazyHTML hashes the combined declaration set and injects one rule into a runtime <style> tag, shared by every element with the same visual signature.

Philosophy

Most utility-class frameworks still make you choose class names and maintain a mapping in your head. LazyHTML removes that layer: the attribute is the declaration, in plain English words rather than abbreviated class tokens, so a designer or a template author can read intent directly off the markup.

Design goal Combine the maximum number of visual attributes on a single element, and let the stylesheet exist only as a byproduct — generated, cached, and deduplicated automatically.

Modifiers — states & breakpoints

Any attribute can be prefixed with a state and/or a breakpoint, separated by colons. Order between prefixes doesn't matter.

htmlmodifiers
<a
  bg="#141d26"
  hover:bg="#34d399"
  md:hover:scale="1.05"
>Link</a>

Laziness

Two layers of "lazy" are at play:

  • Rule laziness: a declaration set is only ever converted into a CSS rule the first time it's seen; every later element with the same attributes reuses the cached class.
  • Viewport laziness: add lazy="true" to an element to defer its resolution until it is about to scroll into view, via IntersectionObserver.
Note A MutationObserver keeps watching the document after the initial scan, so content injected later (routing, fetch()-loaded HTML) is picked up automatically — call LazyHTML.scan(node) manually if you ever need to force it.

JavaScript API

MethodDescription
LazyHTML.scan(root?)Resolves every lazy attribute under root (default: document).
LazyHTML.resolve(el)Resolves a single element immediately.
LazyHTML.registerAttribute(name, fn)Adds a new word to the vocabulary. fn(value, el) returns an array of "prop:value" strings.
LazyHTML.registerState(name, pseudo)Adds a new state modifier, e.g. "open" → ":is([open])".
LazyHTML.registerBreakpoint(name, mediaQuery)Adds a new responsive prefix.
LazyHTML.resolveColor(value)The color-parsing helper used internally by color / bg.

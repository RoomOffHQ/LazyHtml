The attribute
is the stylesheet.

LazyHTML is a dependency-free runtime that reads styling straight off your HTML tags — color, spacing, layout, hover states, breakpoints — and lazily resolves them into one shared, deduplicated CSS class. No build step. No naming decisions. No stylesheet to maintain.

why lazyhtml

Built around one idea, pushed all the way

Combine the maximum number of visual attributes on a single element — everything else follows from that.

Full vocabulary

Around 50 attributes covering typography, box model, layout, position, borders, effects — named in plain English, not abbreviated tokens.

Composable modifiers

Prefix any attribute with a state (hover:, focus:) and/or a breakpoint (md:, lg:), in any order.

Lazy by design

Rules are cached and deduplicated automatically. Mark an element lazy="true" to defer its resolution until it nears the viewport.

Tiny, extensible API

registerAttribute, registerState and registerBreakpoint let you grow the vocabulary without forking the library.

Raw CSS escape hatch

Anything outside the vocabulary is one lz-* attribute away — lz-text-shadow, lz-backdrop-filter, any property at all.

Zero build step

One script tag. No compiler, no CDN dependency, no PostCSS pipeline. Open demo/index.html directly in a browser.

Without LazyHTML

A class name to invent, a rule to write somewhere else, and a mental round-trip every time you tweak a value.

cssstyles.css
.btn-primary {
  background: #34d399;
  color: #04150e;
  padding: 10px 20px;
  border-radius: 999px;
  font-weight: 700;
}
.btn-primary:hover {
  background: #4fe3ac;
  transform: scale(1.03);
}

With LazyHTML

The same result, declared once, on the element it styles. Move it, copy it, delete it — nothing else to update.

htmlindex.html
<button
  bg="#34d399" color="#04150e"
  pad-x="20" pad-y="10"
  radius="999" bold="true"
  hover:bg="#4fe3ac"
  hover:scale="1.03"
>Ship it</button>
